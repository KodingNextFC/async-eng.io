What Is an Asynchronous Function?

Great question! We use asynchronous functions. This way, JavaScript doesn't freeze the web page while waiting for data — it keeps everything running smoothly.

JavaScript can only do one thing at a time. Without asynchronous functions, the entire page would "freeze" every time it waits for something slow, like fetching data from the internet.

Imagine you're ordering food at a restaurant. You don't stand there waiting for the chef to finish cooking — you sit down, chat, or check your phone. Once the food is ready, the waiter calls you over. Asynchronous functions work the same way: JavaScript keeps doing other things until the data is ready.

Getting to Know async and await

The async keyword before a function marks it as running asynchronously. Inside it, we can use await to "wait" for the result of a process — without stopping the entire program.

async function fetchAirQuality() {
  // wait for the server to respond
  const response = await fetch("https://api.example.com/air-quality");
  // convert the response into usable data
  const data = await response.json();
  return data;
}
  1. async marks this function as allowed to use await inside it.
  2. await fetch(...) waits for the server's response, without stopping the page.
  3. await response.json() converts the response into ready-to-use data.
  4. The function returns data so it can be used elsewhere in our code.

Why Does This Matter?

Without asynchronous functions, the "Check Air Quality" button would freeze the entire page until the data finishes loading from the server. With async/await, users can still scroll, click, and interact with the page while the data loads in the background.

Tip: The fetchAirQuality() function only runs when it's called — for example, when a user clicks the "Check Air Quality" button on our web page.
Key Takeaway

Asynchronous functions help JavaScript handle long-running tasks — like fetching data from an API — without stopping the entire program.
